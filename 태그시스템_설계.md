# 합격수기 태그 시스템 설계

## 핵심 철학

- **고정된 것 없음** (카테고리 이름, 태그 이름 모두 유동적)
- AI가 수기를 읽고 자유롭게 태그 생성
- 데이터가 쌓일수록 태그가 자연스럽게 수렴
- 단, 강사/교재는 수업 추천에 직접 쓰이므로 별도 구조화

---

## 태그 동작 방식

### 1단계: 자유 추출
AI가 수기 하나를 읽고 태그를 자유롭게 뽑음
```
수기 읽음 → 태그 목록 반환
예: ["직장병행", "초시생", "40대", "인강위주", "하루8시간",
     "새벽기상", "슬럼프극복", "산책으로멘탈관리", "기출위주"]
```

### 2단계: 기존 태그와 비교
AI가 추출한 태그를 DB에 있는 기존 태그 목록과 비교
- 동일하거나 충분히 유사하면 → 기존 태그 사용
- 새로운 의미면 → 새 태그 추가

### 3단계: 주기적 정규화 (100개마다)
쌓인 태그 중 비슷한 것들을 AI가 통합
```
직장병행 / 회사다니면서 / 직장인수험생 / 재직중 공부
→ "직장병행" 으로 통일 후 DB 일괄 업데이트
```

## DB 구조

### 합격수기 테이블 (passnotes)
| 컬럼 | 내용 |
|------|------|
| id | 고유번호 |
| title | 제목 |
| views | 조회수 |
| url | 원본 URL |
| 응시시험 | 국가직/지방직 등 (summary) |
| 급수 | 7급/9급 등 |
| 응시지역 | |
| 응시직렬 | |
| 수험기간 | |
| 합격년도 | |
| content | 본문 전체 |
| keywords_extracted | AI 처리 완료 여부 (0/1) |

### 태그 테이블 (tags)
| 컬럼 | 내용 |
|------|------|
| id | 고유번호 |
| name | 태그명 ("직장병행", "초시생" ...) |
| count | 사용된 수기 수 (자동 집계) |

### 수기-태그 연결 테이블 (passnote_tags)
| 컬럼 | 내용 |
|------|------|
| passnote_id | 수기 id |
| tag_id | 태그 id |

### 강사 테이블 (instructors)
| 컬럼 | 내용 |
|------|------|
| id | 고유번호 |
| passnote_id | 수기 id |
| subject | 과목명 ("국어", "영어", "한국사" ...) |
| name | 강사명 ("신민숙", "비비안", "이중석" ...) |

### 교재 테이블 (textbooks)
| 컬럼 | 내용 |
|------|------|
| id | 고유번호 |
| passnote_id | 수기 id |
| subject | 과목명 |
| name | 교재명 ("올인원 블랭크노트", "또또북" ...) |

## AI 추출 프롬프트 방향

```
이 합격수기를 읽고:

1. 이 사람을 설명하는 태그를 자유롭게 추출해줘
   - 생활상황, 나이대, 수험이력, 공부방법, 루틴, 멘탈 등
   - 개수 제한 없음, 이름 제한 없음
   - 단, 기존 태그 목록에 비슷한 게 있으면 기존 것 사용

2. 강사 정보: {과목: 강사명} 형태로
3. 교재 정보: {과목: 교재명} 형태로

기존 태그 목록: [...]
```

## 정규화 기준 예시

| 통합 전 | 통합 후 |
|---------|---------|
| 직장병행, 회사다니면서, 직장인수험생, 재직중 | 직장병행 |
| 대학생, 대학교재학중, 학업병행, 재학중 | 대학생병행 |
| 사회경험있음, 직장경험있음, 회사경험 | 직장경험보유 |
| 노베이스, 기초없음, 완전처음 | 노베이스 |
| 새벽기상, 새벽형, 아침형인간 | 새벽형 |
| 산책으로멘탈관리, 운동으로극복, 걷기 | 운동/산책 |

## 태그 활용 예시 (나중에)

```
학생 진단 결과: 직장병행 + 초시생 + 노베이스
→ 세 태그 모두 가진 합격수기 매칭
→ 조회수 높은 순으로 추천
→ 해당 수기의 강사/교재 추천
```
