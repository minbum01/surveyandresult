# 합격수기 탐색 설문조사 설계

## 서비스 목적

사용자가 8개 질문에 답하면, 실제 DB에 있는 2,112개 합격수기 중
"나와 가장 비슷한 사람"의 수기를 필터링해 보여주는 매칭 서비스.

설문 선택지 = DB 태그에 직접 매핑 → 결과 페이지에서 AND 조건 필터링

---

## 전체 화면 흐름

```
인트로 → Q1 → Q2 → Q3 → Q4 → Q5 → Q6 → Q7 → Q8 → 로딩 (2초) → 결과
```

레퍼런스 참고 포인트:
- 인트로: 라이트 테마, 큰 타이포그래피, 임팩트 있는 카피
- 질문: 다크 테마, 가로 스와이프 카드, 배경 이미지 전환
- 로딩: "X만 명의 데이터 분석 중..." 연출
- 결과: 라이트 테마, 수기 카드 목록

---

## 질문 설계 (8문항)

### Q1. 지금 어떤 상황에서 준비하고 있나요?

> 현재 생활 환경을 알려주세요

| 선택지 | DB 태그 |
|--------|---------|
| 공부에만 올인 중 | `전업수험생` |
| 직장 다니면서 준비 | `직장병행` |
| 학교 다니면서 준비 | `재학수험생` |
| 육아하면서 준비 | `육아병행` |
| 은퇴/퇴직 후 준비 | `퇴직수험생` |
| 아르바이트하면서 준비 | `아르바이트병행` |

## 매칭 로직

### 기본 전략: 태그 AND 조건, 점수 기반 정렬

```
필수 조건 (AND): Q1(상황), Q4(베이스)
선택 조건 (점수): Q2, Q3, Q5, Q6, Q7, Q8

각 태그 보유 시 +1점 → 점수 높은 순 정렬
```

이렇게 하면 "0개 매칭" 문제를 피하면서도 가장 비슷한 수기가 위로 올라옴.

### Fallback 전략

- 결과 10개 미만 → Q4(베이스) 조건 추가 해제 후 재검색
- 결과 여전히 5개 미만 → Q1(상황)만 유지, 나머지 해제

## 결과 페이지

```
[분석 완료]
2,112개 합격수기 중 나와 비슷한 수기 47개 발견

내 프로필: #직장병행 #노베이스 #6개월~1년 #인강병행 #멘탈관리

┌─────────────────────────────────────────┐
│ 직장 다니며 6개월 만에 합격한 이야기       │
│ 조회수 12,450  |  2024 지방직 9급         │
│ #직장병행 #노베이스 #인강병행 #멘탈관리    │
│ 일치 태그 4/5                            │
└─────────────────────────────────────────┘

...
```

- 일치 태그 수 + 조회수 가중 정렬
- 카드 클릭 → 수기 본문 + 강사/교재 정보

## 기술 구현 방향

| 항목 | 방법 |
|------|------|
| 화면 | 단일 HTML 파일 (vanilla JS) |
| 스타일 | Tailwind CDN |
| 데이터 | FastAPI `/reviews?tags=직장병행,노베이스` |
| 카드 스와이프 | CSS scroll-snap + 터치 이벤트 |
| 로딩 연출 | 2초 카운팅 후 결과 전환 |

---

## 남은 구현 순서

1. ⬜ FastAPI 백엔드 — `/tags`, `/reviews` API (점수 기반 정렬 포함)
2. ⬜ 설문 HTML — 인트로 + Q1~Q8 + 로딩 + 결과
3. ⬜ 수기 상세 페이지 — 본문 + 강사/교재 목록
