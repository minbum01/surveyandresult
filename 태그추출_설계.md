# 태그 추출 설계

## 방식: Claude Code CLI 서브프로세스

별도 API 비용 없이 Max 구독 안에서 처리.
Python 스크립트가 `claude -p "..."` 명령을 수기마다 호출.

```
Python 스크립트
  └─ DB에서 수기 하나 읽기
  └─ claude -p "[프롬프트 + 수기 본문]" 실행
  └─ 응답(JSON) 파싱
  └─ DB에 태그/강사/교재 저장
  └─ 다음 수기로
```

---

## 프롬프트 설계

```
다음 공무원 합격수기를 읽고 아래 3가지를 JSON으로 반환해줘.

[규칙]
1. tags: 이 사람을 설명하는 키워드를 자유롭게 추출
   - 생활상황, 나이대, 수험이력, 공부법, 루틴, 멘탈관리 등
   - 개수 제한 없음
   - 아래 기존 태그 목록에 같거나 비슷한 게 있으면 반드시 기존 것 사용
   - 진짜 새로운 의미일 때만 새 태그 생성

2. instructors: 언급된 강사 정보
   - {과목: 강사명} 형태
   - 과목은 정확히 (국어, 영어, 한국사, 행정학, 행정법, 헌법 등)
   - 강사명은 성+이름 풀네임으로 정규화 (쌤/T/선생님 제거)

3. textbooks: 언급된 교재/강의 정보
   - {과목: 교재명} 형태

[기존 태그 목록]
{existing_tags}

[합격수기]
제목: {title}
응시시험: {exam_type} {grade} {job_series}
수험기간: {study_period}

{content}

[출력 형식 - JSON만 반환]
{
  "tags": ["태그1", "태그2", ...],
  "instructors": [
    {"subject": "과목", "name": "강사명"},
    ...
  ],
  "textbooks": [
    {"subject": "과목", "name": "교재명"},
    ...
  ]
}
```

## 기존 태그 전달 방식

매 수기마다 현재 DB에 있는 태그 전체를 프롬프트에 포함.
태그가 늘어날수록 AI가 기존 태그를 더 잘 재사용.

```
초반 (1번째): 기존 태그 0개 → 전부 새로 생성
중반 (500번째): 기존 태그 ~150개 → 대부분 재사용
후반 (2000번째): 기존 태그 ~200개 → 거의 재사용, 가끔 신규
```

## 태그 정규화 (100개마다)

100개 처리 후 아래 프롬프트로 정규화 실행:

```
현재 태그 목록이야:
{전체 태그 목록}

의미가 같거나 매우 비슷한 태그들을 묶어서
하나의 대표 태그로 통일해줘.

출력 형식 - JSON만:
[
  {"keep": "직장병행", "remove": ["회사다니면서", "직장인수험생", "재직중"]},
  {"keep": "노베이스", "remove": ["기초없음", "완전처음"]},
  ...
]
```

정규화 결과에 따라 DB passnote_tags 일괄 업데이트.

---

## 재개 기능

DB의 `keywords_extracted` 컬럼으로 진행 상황 관리.
```
0 = 미처리
1 = 완료
```
스크립트 재실행 시 `keywords_extracted = 0`인 것부터 이어서 처리.

## 예상 소요 시간

| 항목 | 계산 |
|------|------|
| 수기 1개당 | ~10초 (claude CLI 호출 + 응답) |
| 전체 2,114개 | 약 6시간 |
| 정규화 (21회) | 약 10분 |
| **총계** | **약 6시간** |

백그라운드로 돌리면 됨.

---

## 오류 처리

- JSON 파싱 실패 → 해당 수기 건너뛰고 계속
- claude CLI 타임아웃 → 재시도 1회, 실패 시 건너뛰기
- 중단 시 → 재실행하면 미완료분부터 자동 재개
