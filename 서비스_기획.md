# 합격수기 탐색 서비스 — 기획서 (최신)

---

## 서비스의 본질

공시 수험생이 이 서비스에 오는 이유는 하나입니다.

> **"나 같은 사람도 합격할 수 있을까?"**

직장 다니면서 공부하는 사람,
아이 키우면서 새벽에만 책 펴는 사람,
완전 노베이스로 시작하는 사람,
학원 가고 싶은데 맞는 방법인지 불안한 사람.

이 서비스는 그 질문에 **"그래, 너 같은 사람이 합격한 기록이 여기 있어"** 라고 답하는 서비스입니다.

---

## 핵심 철학

### 통계는 가능성을 증명하는 도구다

통계를 쓰는 목적:
- ✅ "당신과 같은 조건에서 합격한 사람이 **N명** 있습니다"
- ✅ "그 사람들은 평균 **11개월** 만에 합격했습니다"
- ✅ "그들이 가장 많이 겪은 고민은 **멘탈 관리**였습니다"

통계를 쓰면 안 되는 방식:
- ❌ "합격자 63%가 인강을 했는데 당신은 학원 → 달성도 낮음"
- ❌ "합격자 평균 준비기간 11개월 → 당신 목표 6개월 → 53% 달성"
- ❌ "대부분은 이랬는데 당신은 다름 → 위험"

**어떤 선택도 틀리지 않습니다.**
학원을 다니면서 합격한 직장인도 있고,
독학으로 6개월 만에 붙은 노베이스도 있습니다.
서비스가 할 일은 그 경로의 선례를 찾아주는 것이지,
다수의 경로를 정답으로 강요하는 게 아닙니다.

### 개인화는 선택의 존중이다

8개 질문으로 사용자의 선택을 모읍니다.
그 선택 하나하나는 **필터이자 존중받아야 할 조건**입니다.

"직장 다니면서 학원 가고 싶다" → 그 조합으로 합격한 사람을 찾아준다.
"그건 힘든 길이야, 인강이 낫지 않아?" 라고 말하는 서비스가 아닙니다.

---

## 설문 8문항의 목적

설문은 DB 필터 태그를 수집하는 과정입니다.
각 답변은 DB 태그에 매핑되어, 가장 유사한 합격수기를 찾는 조건이 됩니다.

| 질문 | 목적 |
|------|------|
| Q1. 현재 상황 | 핵심 필터 — 같은 환경의 합격자 추출 |
| Q2. 목표 시험 | 시험 종류 매칭 |
| Q3. 목표 직렬 | 과목 구성 참고 |
| Q4. 베이스 | 출발점이 같은 합격자 추출 |
| Q5. 목표 기간 | 기간 현실성 판단용 (판단하되 강요하지 않음) |
| Q6. 하루 시간 | 학습량 참고 |
| Q7. 공부 방식 | 방식이 같은 합격자 추출 (이 방식으로 된다는 증명) |
| Q8. 고민 | 같은 고민을 극복한 합격자 추출 |

## 결과 페이지 목적

> "2,112개 합격수기 중 당신과 가장 비슷한 사람들을 찾았습니다.
> 그들이 어떻게 합격했는지 보여드립니다."

사용자가 결과를 보고 나서 느껴야 할 감정:

1. **"나 같은 사람이 있구나"** — 고립감 해소
2. **"이 사람들도 됐네"** — 가능성 확인
3. **"이렇게 하면 되겠다"** — 방향 획득
4. **"빨리 시작하고 싶다"** — 동기 생성

느끼면 안 되는 감정:

- ~~"나는 기준 미달이네"~~
- ~~"내 선택이 틀린 건가"~~
- ~~"이 사람들처럼 안 하면 떨어지겠다"~~

## 결과 페이지 섹션 재설계

### [r-report] 레포트 카드 — 왼쪽 전체

**역할**: "당신이 누구인지, 그리고 당신과 비슷한 합격자가 몇 명인지"

- 페르소나 타입 (야행성 헌터, 시간의 마법사 등)
- 핵심 지표 4개: 유사 합격자 수 / 평균 준비기간 / 초시 합격률 / 종합 점수
- 분석 텍스트 5줄: 당신의 조건 요약 + 합격 가능성 확인 메시지
- 내 선택 태그 목록

**기조**: "당신과 비슷한 X명이 합격했습니다" — 가능성 증명

---

### [r-diag] 진단 점수 → **"이 선택으로 합격한 사람들"**

**역할**: 내 선택 조합(Q1+Q4+Q7)으로 합격한 사람들의 실제 모습

기존 문제: 내 답변을 점수로 변환 → 자기참조, 의미 없음
새 방향: 내 선택 조합과 동일한 합격자들의 핵심 데이터를 보여줌

표시 내용:
```
직장병행 + 노베이스 + 학원 선택 → 합격자 N명

  평균 준비 기간   13.4개월
  초시 합격률      54%
  평균 하루 순공   6.8시간
  가장 많이 언급   토요일 전일반 루틴
  공통 고민        멘탈 관리 (71%)
```

**기조**: "당신의 선택으로 합격한 사람들이 있고, 그들은 이랬습니다"

---

### [r-real] 목표 현실성

**역할**: 기간 목표가 같은 상황 합격자 분포에 비춰 현실적인지 체크

- 별점 + 판정 레이블 (충분히 가능 / 빡빡함 / 재설정 권고)
- 같은 상황 합격자 기간 분포와 내 목표 비교
- 점수가 낮아도 "불가능"이 아닌 "이렇게 준비하면 가능" 메시지

**기조**: 현실을 직시하되 가능성은 열어둠

---

### [r-notes] 나와 비슷한 합격수기

**역할**: 태그 매칭된 실제 합격수기 카드 목록

- 가장 많이 일치하는 태그 수기가 앞으로
- 카드에 일치 태그 수 / 준비 기간 / 조회수 표시
- 클릭하면 수기 본문으로 이동

**기조**: "이 사람이 나랑 비슷하네" 공감 유발

---

### [r-style] 학습 스타일 적합도

**역할**: 선택한 공부 방식이 현재 상황에서 어떻게 작동하는지 안내

- 같은 방식 + 같은 상황으로 합격한 선례 유무
- ✅ 적합 / ⚠ 주의할 점 / 주의 시에도 "이렇게 하면 됨" 대안 제시

**기조**: 선택을 바꾸라는 게 아니라, 선택이 통하는 방법을 알려줌

---

### [r-risk] 핵심 리스크

**역할**: 같은 상황 + 같은 고민을 가진 수험생들이 실제로 겪은 실패 패턴과 대응

- 비슷한 사람들의 공통 실패 원인 2가지
- 각각 구체적인 대응 방법

**기조**: 겁주는 게 아니라 "이 함정을 미리 알면 피할 수 있다"

---

### [r-instr] 강사 추천

**역할**: 같은 상황 합격자들이 가장 많이 선택한 강사

- TOP 3 + 선택 비율 바 차트
- 📌 DB 연결 필요

---

## 데이터 활용 원칙

1. **비교의 기준은 항상 "나와 같은 상황의 합격자"**
   전체 합격자 평균이 아닌, Q1(상황) 필터링된 합격자만 사용

2. **통계는 가능성 증명에만 사용**
   "N명이 이 방식으로 합격했다" O
   "합격자 다수와 다른 방식이다" X

3. **소수 경로도 유효하다**
   선택한 방식으로 합격자가 적더라도, 있다면 보여준다
   없다면 "가장 유사한 조합"으로 fallback

4. **점수는 판단이 아닌 참고**
   현실성 점수가 낮아도 "불가능"이 아닌 "더 준비가 필요한 부분"으로 표현
