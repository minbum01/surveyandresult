# 합격수기 탐색 서비스 - 최종 구현 설계

## 서비스 개요

공무원 합격수기 2,114개를 AI로 분석해 태그/강사/교재를 추출하고,
학생이 자신과 비슷한 수기를 찾아보고 강사·교재 추천까지 받는 서비스.

## DB 구조 (현재)

```
passnotes         - 합격수기 본문
  id, title, views, url
  exam_type, grade, region, job_series
  study_period, pass_year, category
  content, keywords_extracted(0/1)

tags              - 태그 목록
  id, name, count

passnote_tags     - 수기 ↔ 태그 (다대다)
  passnote_id, tag_id

instructors       - 수기별 강사
  id, passnote_id, subject, name

textbooks         - 수기별 교재
  id, passnote_id, subject, name
```

---

## 태그 추출 방식 (extract_tags.py)

1. `keywords_extracted = 0`인 수기부터 순서대로 처리
2. Claude CLI에 수기 본문 + 기존 태그 목록 전달
3. Claude가 JSON 반환: `{ tags, instructors, textbooks }`
4. 기존 태그와 유사한 건 기존 것 재사용, 새 의미만 신규 추가
5. 100개마다 정규화 (유사 태그 자동 통합)
6. 중단 후 재실행해도 이어서 처리

## API 설계 (백엔드)

```
GET /tags
  → 전체 태그 목록 + 빈도

GET /reviews?tags=노베이스,인강병행&sort=views
  → 태그 AND 조건으로 수기 목록

GET /reviews/:id
  → 수기 본문 + 강사 목록 + 교재 목록

GET /instructors/top?subject=국어
  → 과목별 강사 언급 횟수 순위

GET /tags/groups
  → 카테고리별 태그 그룹 (AI로 분류)
```

## 강사 추천 로직

태그 매칭된 수기들의 강사 데이터를 집계해 추천:

```

국어: 신민숙(21회) > 이선재(8회) > 김대현(3회)
→ "신민숙 선생님이 가장 많이 추천됨"
```
