## 0. 작성 원칙 (Hard Rules)

1. **정확한 수치(%)·구체적 명수 금지**
   - ❌ "합격률 67%", "초시 72%", "유사 합격자 487명", "평균 9.8개월"
   - ⭕ "다수의 합격자가...", "보통 1년 안팎의 준비 기간이 일반적", "비슷한 분들 중 절반 이상이..."
   - 예외: 사용자가 직접 입력한 값(없음 — 본 설문엔 자유 입력 없음)

2. **부정 워딩 금지 → 긍정 프레이밍 통일**
   - ❌ 어렵다 / 힘들다 / 위험하다 / 약점 / 약하다 / 리스크 / 포기 / 실패 / 도전적 / 빡빡한 / 무리한 / 조심하세요
   - ⭕ 보완 영역 / 성장 포인트 / 성공 키 / 체크 포인트 / 이렇게 하면 더 빠릅니다 / 한 단계 끌어올리는 법

3. **사용자 답변 기반 개인화 필수**
   - 모든 콘텐츠는 Q1~Q8 답변 조합에 따라 다른 텍스트가 노출되어야 함
   - 누구에게나 동일한 일반론은 지양

## 1. 데이터 입력

설문 종료 시 `sessionStorage.passNoteAnswers` 에 JSON 저장됨.

```js
// 형태
[
  [{tagId:'전업수험생', label:'전업 수험생'}],          // Q1
  [{tagId:'국가직9급', label:'국가직 공무원'}],         // Q2
  [{tagId:'행정직',     label:'일반행정직'}],            // Q3
  [{tagId:'노베이스',   label:'공부는 처음이야!'}],      // Q4
  [{tagId:'1년이내',    label:'1년 이내'}],              // Q5
  [{tagId:'하루10시간이상', label:'10시간 이상'}],       // Q6
  [{tagId:'기출반복', ...}, {tagId:'인강병행', ...}],    // Q7 (1~2개)
  [{tagId:'멘탈관리',   label:'멘탈 관리'}],             // Q8
]
```

result.html 진입 시 위 데이터를 읽어 섹션을 렌더.
데이터가 없으면 `index.html` 로 부드럽게 리다이렉트.

## 3. 페이지 섹션 구성 (Top → Bottom)

### Section 1 — 페르소나 카드 (Hero)
**입력**: Q1
**목적**: 사용자에게 "당신은 누구인가" 를 한 줄로 명명. 첫인상에서 동질감/자기인식 부여.
**구성**:
- 페르소나 칭호 (Q1별 1개)
- 한 줄 동기 메시지 (Q1×Q4 변주)
- 상황·시험·직렬·베이스 태그 칩 (Q1, Q2, Q3, Q4)

**페르소나 매핑**

| Q1 | 칭호 | 키 메시지 톤 |
|---|---|---|
| 전업수험생 | 풀타임 러너 | 시간이 곧 자산 |
| 직장병행 | 퇴근 후 헌터 | 절박함이 효율을 만듭니다 |
| 재학수험생 | 캠퍼스 챌린저 | 흡수력의 황금기 |
| 육아병행 | 새벽의 마법사 | 작은 시간을 모아 만드는 기적 |
| 퇴직수험생 | 인생 2막 도전자 | 풍부한 경험이 곧 강점 |
| 아르바이트병행 | 멀티태스커 | 한 시간 한 시간이 빛나는 사람 |

**예시 카피 (전업수험생 + 노베이스)**
> 풀타임 러너
> 백지에서 시작해도 시간을 가장 진하게 쓸 수 있는 분.

### Section 2 — 나의 출발점 (Identity)
**입력**: Q1 + Q4
**목적**: "어디서 출발하는지" + 그 출발점의 강점을 짚어줌.
**구성**:
- 한 줄 요약: "[Q1 상황]에서 [Q4 베이스]로 출발하는 분"
- 강점 3가지 (Q4 기준 긍정 항목)
- "비슷한 출발점에서 합격한 분들도 많습니다" 사회적 증거 한 줄

**Q4별 강점 프레이밍**

| Q4 | 강점 카피 |
|---|---|
| 노베이스 | 선입견 없이 가장 효율적인 길로 갈 수 있는 자유 / 새 습관을 빠르게 정착시킬 수 있는 백지 상태 |
| 수능베이스 | 국어·영어 기초 감각 보유 / 장기 학습 루틴 경험 보유 |
| 전공자 | 전공 과목에서 시작부터 우위 / 비전공 과목에 시간을 몰아줄 여유 |
| 토익베이스 | 영어 부담 절약 / 다른 과목에 시간 집중 가능 |

> 주의: "이과 출신이라 국어가 약해요" 같은 약점 부각 표현은 ❌

### Section 3 — 추천 경로 (Path Forward)
**입력**: Q1 + Q5 + Q6
**목적**: "비슷한 분들이 가장 많이 따라간 길" 을 보여주며 자신감 부여.
**구성**:
- 추천 페이스 한 줄: "[Q1] 분들 중 [Q5]·[Q6] 조합으로 합격한 분들의 경로"
- 단계별 마일스톤 3~4단계 (Q5에 따라 다름)
  - 1년 이내: 기초 다지기 → 기출 정복 → 실전 마무리
  - 1년 이상: 기초 완성 → 기출 마스터 → 심화 + 실전 → 마무리
- 각 단계 핵심 활동 (Q4·Q7로 변주)

**예시 (직장병행 + 1년 이내 + 10시간 미만)**
> "퇴근 후 시간을 가장 효율적으로 쓰는 길"
>
> 1단계 — 기초 압축
>   강사 1명을 정해 압축 인강을 1.5배속으로. 매일 같은 시간대 고정.
> 2단계 — 기출 집중
>   주말은 기출 풀이, 평일은 오답 정리.
> 3단계 — 실전 감각
>   주 2회 모의고사 루틴. 단권화 노트 마무리.

### Section 4 — 학습 전략 (Strategy)
**입력**: Q4 + Q7
**목적**: 베이스와 선호 방식을 결합한 맞춤 학습법 제시.
**구성**:
- 한 줄 전략 헤드라인
- 추천 우선순위 (베이스의 강점 살리기)
- Q7에서 고른 방식의 결합 팁

**전략 카피 (Q4×Q7 매트릭스 일부)**

| Q4 \ Q7 | 인강+기출 | 기출+회독 | 독학+스터디 |
|---|---|---|---|
| 노베이스 | 강사 커리큘럼을 끝까지 따라가며 기출은 매일 일정량 | 회독은 핵심 단원만, 기출 비중 절반 이상 | 스터디로 외로움 보완, 독학 루틴 사수 |
| 전공자 | 전공은 빠르게, 비전공에 인강·기출 시간 몰아주기 | 전공은 회독 1회, 비전공 기출 위주 | 전공 강점으로 스터디 리더 역할 |
| 토익베이스 | 영어는 점검 수준, 다른 과목 인강·기출에 집중 | 영어 외 과목 회독·기출 균형 | 영어 강점으로 스터디 기여, 나머지에 시간 투자 |

## 4. 톤 & 워딩 가이드

### 어조
- 합쇼체 또는 평어 혼용. "~합니다", "~해보세요" 위주.
- 친근하지만 전문가스러운 균형.
- 1인칭 호칭은 "당신". "고객님", "수험생님" 지양.

### 강조 패턴
- 굵은 글씨로 핵심 단어 1~2개씩 강조 (`<strong>`)
- "비슷한 분들은 / 가장 많이 / 이렇게 풀어냈습니다" 같은 사회적 증거 표현 적극 활용

### 자주 쓰는 표현 (DO)
- "비슷한 분들 중 다수가..."
- "가장 일반적인 길은..."
- "보통 ~ 정도의 페이스가 ~"
- "한 분이 한 분에게 추천한다면..."
- "이렇게 시작한 분들이 ~"
- "강점을 살려 ~"
- "한 단계 끌어올리는 법"

### 금지 표현 (DON'T)
- "어렵습니다 / 힘듭니다 / 위험합니다 / 약합니다"
- "리스크 / 약점 / 단점 / 한계"
- "포기하지 마세요" (포기라는 단어 자체 금지)
- "실패 / 좌절"
- "조심하세요 / 주의하세요"
- "너무 / 매우" 같은 감정 강도어 (필요시 한 번만)
- "67%, 487명, 9.8개월" 같은 정확 수치 (도메인 데이터로 검증 불가하면 절대 노출 금지)

### 수치 표현 우회법
- "~의 합격자 중 다수가" (퍼센트 대신)
- "비슷한 조건의 합격자가 적지 않게" (구체 명수 대신)
- "보통 1년 안팎" (개월 수 대신)
- "절반 이상이 / 가장 많이" (특정 % 대신, 추세만)

| 5. 루틴 | Q1, Q6 | 12 조합 |
| 6. 과목 순서 | Q4, Q3 | 4 × 7 = 28 조합 |

> 모든 조합을 일일이 카피로 적기보다는 **템플릿 + 변수 치환** 방식 권장.
> 변수: 페르소나 칭호, 시간대, 강점 키워드, 액션 동사 등.

## 6. 레이아웃 권장

- **세로 스크롤 형 리포트** — 출력(인쇄) 친화적 흐름
- **A4 인쇄 호환** — `@media print` 룰로 1~2페이지 분량 정돈
- **카드/벤토 그리드** — 섹션별 비주얼 구분
- 강조 컬러 1~2개 (예: 골드, 다크블루) — 합격 리포트의 격(格) 표현
- 큰 페르소나 헤더 + 섹션마다 큰 번호 (Editorial 톤)

> 기존 `index.html` 안의 `screen-result` 그리드(현재 dead code)는 참고용으로 유지하되, **새 result.html 은 폰트·색·레이아웃을 처음부터 새로 디자인** 권장 (현재 result.html 의 따뜻한 베이지/네이비 팔레트가 출발점으로 적합).
